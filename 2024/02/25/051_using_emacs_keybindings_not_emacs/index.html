Using Emacs-like Keybindings, not Emacs

Junhao Wang
2024-02-25
Emacs
Emacs

I am writing this post to share some thoughts about how I have been using Emacs-like keybindings in my Mac and Windows machines. I used Emacs in the past but now I am doing most of my development work in CLion, Visual Studio, and Vscode. I use Emacs occasionally when I want to quickly edit some files (e.g. config files).

Over time I realized that I would prefer using IDEs on projects rather than being a pure Emacs user. In addition, I developed my own keybinding schema that I really enjoyed, which came from experimentations and other people’s ideas from the Internet.

What is Emacs and Why Not?

Emacs is a popular text editor designed for POSIX operating systems and it is available on Linux, macOS, and Windows. We love Emacs because it is a powerful “system” that enables users setting up various commands to complete complex actions (keymaps).

Interestingly, almost all my friends and colleagues use Vim, not Emacs. I used Vim occasionally to edit config files, especially when I am working with a new EC2 instance that does not have a lot of stuff installed. Unfortunately, I decided to use Emacs very long ago and I still sticked to its keybindings that heavily depend on Ctrl and Alt.

Vim is a very powerful tool for processing texts. Emacs seems to be more than that and some people joke about saying Emacs is an “operating system” (no!) because it is extensible by Lisp and can do a lot of work that a text editor cannot do. In Chinese forums, people said Vim is the god of text editor, while Emacs is the editor of the god (which implies that it is a text editor used by the god). And indeed, using Emacs needs ten fingers (another joke about Emacs).

Here are the reasons why I stopped using the Emacs app:

  1. Configuration takes time. To use Emacs efficiently, you have to put lots of efforts learning from others’ .emacss and integrate the lines you want to your own config file. Installing extensions (or plugins) is not straightforward.
  2. Navigation is important in large projects. Graphical interfaces integrated in IDEs just make life much easier.
  3. Keybingings are only available in Emacs. Over time, I realized I wanted to use Emacs keybindings in other apps such as Chrome, Microsoft Word, Notion, etc. But apparently, Emacs cannot extend to that level.
  4. Emacs is not installed by default. This happened quite often during work when I had to set up a new burner cloud desktop that is running on an EC2 instance. It did not come with Emacs by default. Plus installation, I figured out how to upload my config files as well. Sigh.

In the below sections, I will share how I use Emacs keybindings and some additions I came up with. In the end, I also share how it can be achieved on Windows 11. I use Windows 11 at work because I have to work on game engine code in Visual Studio.
